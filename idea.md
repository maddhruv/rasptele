Build a self-hosted Telegram bot that runs as a Docker container on my Raspberry Pi 5 and serves as the primary control plane and UI for the entire server, so I can monitor and operate everything from my phone without exposing any inbound ports (the bot uses outbound long-polling only). It must be locked to my Telegram user ID so only I can issue commands, load its bot token and service API keys from a .env file (never committed), and require a two-step confirmation button for any destructive action (reboot, container stop, delete). It should push proactive alerts (disk >90% full, container crashed or unhealthy, high CPU temp/throttling, a service going down, new device joining the network), answer on-demand queries for system stats (CPU, RAM, temperature, disk usage) and per-service status (Pi-hole queries/blocks, qBittorrent download progress, Jellyfin activity, Coolify deploy status), and let me take actions through inline keyboards (restart a container, pause/resume downloads, trigger a Jellyfin library scan, hand a magnet link to qBittorrent, approve/deny an unblock request). Architecturally it talks to the host via the Docker socket and psutil for system metrics and to each service through its HTTP API, deploys via Coolify from my GitHub repo for continuous deployment, and is built to grow incrementally — starting with a v1 of stats plus disk/container alerts, then adding one command set per service as each phase comes online.

# Get support for Rasptele

Use the channel that matches the problem so maintainers and other users receive the context they need.

## Ask a usage question

Search the [documentation](README.md#documentation) and existing issues first. If the answer is not available, use the [support question form](https://github.com/maddhruv/rasptele/issues/new?template=support_question.yml) with:

- The Rasptele version or commit
- Raspberry Pi model and operating system
- Deployment method: Docker Compose, Coolify, or Portainer
- Sanitized configuration relevant to the problem
- Exact error text and relevant logs
- Steps already attempted

Remove Telegram tokens, user IDs, Pi-hole passwords, session IDs, public IP addresses, and other secrets.

## Report a reproducible bug

Use the [bug report form](https://github.com/maddhruv/rasptele/issues/new?template=bug_report.yml). Include the smallest reproduction and indicate whether the problem also occurs on the latest release.

## Propose a feature

Use the [feature request form](https://github.com/maddhruv/rasptele/issues/new?template=feature_request.yml). Describe the user problem before proposing an interface or implementation.

## Report a security issue

Do not disclose suspected vulnerabilities in an issue. Follow the [security policy](SECURITY.md) and submit the report through GitHub's private vulnerability-reporting form.

Support is provided by project maintainers and community volunteers without a guaranteed response time.
